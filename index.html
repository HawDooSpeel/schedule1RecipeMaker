To save rules or a preset you can export them and put them somewhere safe because this will not save what you do.

To edit a preset the order of data is names, followed by the text STARTINGPRICESTART, followed by starting item prices, followed by  EFFECTSTART, followed by effects, and then effect price multipliers. Don't use spaces or special characters and put a comma between every effect and item. You can have more effects than items but not more items than effects. All items need an effect but the effect does not need to be unique, so if an item doesn't have an effect just use none as its effect. Put the effect and item in the same spot (the first item's effect needs to be the first effect).

Example: with items 1 2 3 and 4. Items 1 and 3 have effect A and 4 has effect B

1,2,3,4STARTINGPRICESTART0,0EFFECTSTARTA,none,A,B,0,0,0,0

Prices are not required, if you don't want to use them you can just use 0 for all of them.

To save rules or a preset you can export them and put them somewhere safe because this will not save what you do.

Exporting rules follow the same basic rules as exporting names and effects. The main difference is that you don't need to declare the end of a section and there are 4 lists.

The first list is what is being added, the second is what is already there, the third is the result and the fourth is whether the rule is exclusive. All lists need to be the same length

Example: if you add items A B or C to effects D E F and none of them are exclusive you get effects 1 2 or 3 this would be stored as
A,B,C,D,E,F,1,2,3,false,false,false
